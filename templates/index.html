CRUCIBLE SIGINT — ANALYST GUIDE

WHAT IS CRUCIBLE SIGINT?

CRUCIBLE is a passive OSINT infrastructure fingerprinting engine built on the methodology described by Ryan McDonald (Principal Security Engineer | USMC 0341) in his analysis of the DSJ Exchange / BG Wealth Sharing LTD pig-butchering operation — a $150M crypto fraud that victimized thousands of people.

▸ METHODOLOGY CREDIT — RYAN McDONALD
"Fingerprinting Malicious Infrastructure Using Free Resources"
Principal Security Engineer | USMC 0341  ·  Published LinkedIn, May 2026
Ryan demonstrated — using only free passive sources (crt.sh, urlscan.io, DNS, WHOIS, manual JS inspection) — how a single confirmed-bad domain maps an entire criminal infrastructure cluster of 47+ domains, identifies wallet drain mechanisms, and reveals the two-layer brand structure of a $150M pig-butchering operation.
CRUCIBLE automates that methodology. The investigative framework is Ryan's work. This tool is the automation layer built on top of it.

It automates the 5-stage passive pivot Ryan performed manually: certificate transparency → DNS → IP intelligence → WHOIS/RDAP → threat scoring. Version 5 adds urlscan.io corroboration, live JS bundle scanning for wallet drain patterns, cert issuance timelines, and a 12-signal weighted threat model.

All API calls are server-side (Python/FastAPI). Zero CORS issues. No browser restrictions. No API keys required.

WHAT IT DOES — 7-STAGE PIPELINE

  • Stage 1 — Cert Transparency (crt.sh + certspotter): Queries certificate transparency logs for all TLS certificates ever issued for the seed domain and its subdomains. Finds sister domains sharing the same cert infrastructure. Flags NEIBU (内部 = "internal") subdomains — a near-certain Chinese-dev admin panel tell.
  • Stage 2 — DNS Resolution: Resolves all record types (A, AAAA, MX, NS, TXT, CNAME, SOA) via Google DNS-over-HTTPS. Reveals nameserver patterns, MX infrastructure, SPF records, and parking indicators.
  • Stage 3 — IP Intelligence: Enriches resolved IPs with ISP, ASN, country, and hosting type detection. Flags Chinese cloud providers (Alibaba, Tencent), CDN masking (Cloudflare, AWS Global Accelerator), and scam-kit hosting ASNs.
  • Stage 4 — RDAP Registration: Pulls domain registration data including registrar, creation date, status flags, and nameservers. Flags scam-kit registrars (gname.com, nicenic) and fresh domains (<90 days).
  • Stage 5 — urlscan.io Corroboration: Checks urlscan.io for historical scans of the domain. High scan count = well-known bad domain. Also reveals additional IPs and ASNs from browser-based scanning.
  • Stage 6 — JS Bundle Scan: Fetches the live domain and linked JS bundles, scanning for Ryan's key finding: MAX_UINT approve(0xffff...ffff) in deposit flows, ERC-20/TRC-20 wallet interfaces, and iOS .mobileconfig bypass patterns.
  • Stage 7 — Threat Score: Computes a 12-signal weighted composite score with confidence rating. Escalation floors prevent confirmed operations from scoring artificially low due to partial data.

HOW TO INTERPRET RESULTS

Threat Score

0–39 · LOW — insufficient signals or clean domain
40–69 · MEDIUM — suspicious, investigate further
70–100 · HIGH — confirmed or likely malicious

Confidence (HIGH/MEDIUM/LOW) indicates how many data sources contributed to the score. A HIGH score with LOW confidence means one strong signal fired but others couldn't be evaluated — worth investigating but don't share without corroboration.

Key Signals

  • NEIBU ADMIN PORTAL [weight 3.0]: Near-certain scam-kit tell. "neibu" (内部) = internal in Chinese. Admin panel domains for the operation.
  • JS WALLET DRAIN [weight 3.0]: MAX_UINT approve() = persistent theft mechanism. Approval survives domain takedowns.
  • KNOWN OPERATION MATCH [weight 3.0]: Domain matches documented DSJ/BG Wealth Sharing infrastructure fingerprint.
  • SCAM NAMING PATTERN [weight 2.5]: Short random prefixes (dsj*, ffs*, ge7*), suspicious TLDs (.cc, .top), or known scam keywords.
  • DOMAIN VOLUME [weight 2.0]: 10+ domains from CT = scam-kit-as-a-service. Single operators running 47+ frontends with admin-panel-managed rotation.

What Registrars Mean

  • gname.com Pte. Ltd., nicenic: Singapore/China-based, Chinese-developer-preferred. Primary scam-kit registrar — very high signal.
  • Hostinger, Namesilo, Namecheap: Common on funnel/recruitment domains. Medium signal — present in the DSJ operation but not definitive alone.

EFFECTIVE INVESTIGATION WORKFLOW

  • Start with a confirmed-bad seed. One domain from a victim, a report, or law enforcement disclosure. The pivot works from that single seed.
  • Let crt.sh build the cluster. The wildcard query (%.domain.com) surfaces all subdomains and often reveals sister frontends sharing the same cert infrastructure. The full DSJ cluster of 47 domains was found this way.
  • Check NEIBU domains immediately. Any domain flagged NEIBU is an admin panel. Those are the most sensitive findings — they reveal the operator layer, not the victim-facing frontend.
  • Two-layer brand structure: Ryan's key insight — scam operations have a recruitment brand (BG Wealth Sharing) AND a trading platform brand (DSJ Exchange). Investigate both. Finding only one layer = finding half the operation.
  • Re-run weekly. What goes NXDOMAIN = successful takedown. WHOIS updates on the same day you check = operator is still active and watching.
  • Victim immediate action: If MAX_UINT drain detected, direct victims to revoke.cash immediately. The approval persists on-chain after every domain dies.
  • Defang before sharing. Use the DEFANG toggle on all exports. Standard TLP convention: dots → [.], :// → [://].

MODES

  • Standard: Full 7-stage pipeline. Use for initial investigation of any suspected malicious domain.
  • Investigator: Verbose output, raw API responses in JSON export. Use when you need full evidence chain for a report or LEA referral.
  • Phishing / Brand Abuse: Point at YOUR domain to find lookalikes in cert transparency. Returns risk score, live DNS check, and age of each lookalike.
  • Cert Intelligence: Direct crt.sh query with wildcard support. Use for subdomain enumeration or historical cert pattern analysis.
  • Bulk IOC: Paste up to 50 domains/IPs. Enriches each with DNS, IP intel, and RDAP. Exports SIEM-ready CSV.

TECHNICAL NOTES

  • APIs used (all free, no auth): crt.sh (CT logs), certspotter.com (CT fallback), freeipapi.com (IP intel), dns.google (DoH), rdap.org (registration), urlscan.io (historical scans)
  • crt.sh is flaky — it runs on a single Postgres instance and goes down under load. certspotter.com is the fallback. If both fail, retry in 5–10 minutes.
  • JS scan limitations — domains behind Cloudflare bot protection may block the fetch. The scan is opportunistic, not guaranteed.
  • Rate limits — freeipapi.com and urlscan.io have rate limits. The tool includes 300ms delays between IP lookups to respect them.
  • No active probing — CRUCIBLE never sends exploit payloads, port scans, or authenticated requests. All data comes from passive sources and the live domain's public HTTP interface.
